Project Goal:
A full stack project, which aims to develop a website, which allow user to customize their diet plan, in differnet mode(loss weight, muscle build, illness recover, ...)

General workflow:
1. User can to select their diet style first(muscle building/weight lossing/illness recover/...), based on their selected style, 
    the website can give differnet ingredient combonaiton(like more protein for muscle building, less fat for weight lossing...)

2. User can input their Weights, age, gender(which is used to calculate the nutrition they need each day based on their diet style), or they can input their desired 
    daily nutrition/calories manually

3. User can see the nutrition they need each day and calories estimated they need, then the user should be promoted to cutomize their diet(their favorable food/nutrition source), 
then we complete the diet based on their nutrition needs(for example, the user might input chicken chest, rice, and so on to make sure these foods are in their diet, but
we may/may not need futher food material to build a valide diet plan), after the user selected their desired food, we first propose three different set of food material, 
the user are able to choose from those sets to confirm their daily food ingredient

4. After the user selecting their desired food ingredient, we propose a recipe for them, which use the ingredient they selected,


Functionality:

User_system:
1. login: 
    a.login with user email and password

    sign up with their email and password, including password validating,

3. Account setting:

4. General:

tech stack:
1️⃣ High-Level Tech Stack (TL;DR)

Goal: “Full-stack fitness meal planner that generates macro-balanced meal plans and maps them to Coles/Woolies products.”

🌐 Frontend

Framework: React + TypeScript (using Next.js 14+ App Router)

UI: Tailwind CSS + a component library (e.g. shadcn/ui or MUI)

State: React Query (TanStack Query) for server state, basic local state with hooks

Forms & Validation: React Hook Form + Zod

Build/Bundle: Next.js (no extra bundler needed)

Testing: Jest + React Testing Library + Playwright (optional e2e)

🧠 Backend (API)

Runtime: Node.js (LTS)

Framework: Express (or NestJS if you want “enterprise” style)

Language: TypeScript

API style: REST (optional: add GraphQL later)

ORM: Prisma (TypeScript + works nicely with Postgres)

Validation: Zod / Joi on request schemas

Auth: JWT-based auth (Access + Refresh) using passport-jwt or custom middleware

External APIs:

Nutrition: Edamam / Spoonacular

Supermarket data: RapidAPI/Apify Coles/Woolworths (optional)

🗄 Database & Storage

Main DB: PostgreSQL on AWS RDS

Caching (optional): Redis (Elasticache) for hot data (e.g. frequently used recipes)

File Storage: AWS S3 for user avatars / exports

☁️ AWS Infrastructure

Backend hosting: ECS Fargate (Dockerized Node app) in a private subnet

Frontend hosting:

Option A: Next.js static export → S3 + CloudFront

Option B: Deploy Next.js as a Node app on ECS too

Database: Amazon RDS (PostgreSQL), private subnet, no public access

Networking:

VPC with public + private subnets

Application Load Balancer (ALB) in front of ECS service

Security:

AWS Secrets Manager for DB/API keys

IAM roles for ECS tasks to access S3/Secrets

Monitoring:

CloudWatch logs + metrics (CPU, memory, 5xx)

Domain & TLS:

Route 53 for DNS

ACM for free SSL certificates (HTTPS on ALB & CloudFront)

🔁 CI/CD

Git hosting: GitHub

CI/CD: GitHub Actions

Build: Docker for frontend & backend

Deploy:

Push images to Amazon ECR

Update ECS service via GitHub Actions (or AWS CodeDeploy)

Sync built frontend to S3 + invalidate CloudFront cache

5️⃣ CI/CD Pipeline (GitHub Actions → AWS)

High level:

🧪 CI (on every PR / push to main)

Run npm install (or pnpm/yarn) in root.

Build frontend: npm run build --workspace web

Build backend: npm run build --workspace api

Run tests:

npm test --workspace web

npm test --workspace api

Lint: eslint + typescript --noEmit

If all pass → allow merge.

🚀 CD (on push to main)

Backend deploy flow:

Build Docker image for api.

Tag and push to Amazon ECR.

Update ECS task definition with new image tag.

Trigger ECS service deployment (rolling update).

Wait for ECS service to become healthy.

Frontend deploy flow:

Build Next.js app (npm run build --workspace web).

Export static assets (if using static export) or build SSR image:

Static: next export → output to out/

Upload out/ to S3 bucket.

Invalidate CloudFront cache so users get the new version.

You can do all of this from a single deploy.yml workflow.
